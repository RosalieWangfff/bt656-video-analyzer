# BT.656 数字视频流异常数据检测工具

基于 ITU-R BT.656 4:2:2 规范的 10-bit 数字视频流分析工具，用于检测视频数据中的**范围越界**和**SMPTE 彩条阶梯异常**。

## 功能特性

| 功能 | 描述 |
|------|------|
| **10-bit 数据解包** | 支持标准 Packed 4:2:2 (4样本→5字节) 和 Raw 10-bit 两种模式 |
| **SAV/EAV 同步检测** | 自动识别 `3FF 000 000 XYZ` 时序码，解析 F/V/H 控制位 |
| **三级错误判定** | 禁止值(红) / 越界(橙) / 阶梯错误(黄) 三种错误类型 |
| **SMPTE 彩条阶梯校验** | 8级亮度阶梯电平检测，可配置误差容限 |
| **样点序号校准** | 支持手动偏移量，适配任意起始位置的数据片段 |
| **多分辨率支持** | 720×480i, 1280×720p, 1440×1080i, 1920×1080p/i |

## 错误判定规则

工具采用三级错误判定体系，按严重程度递减：

### 1. 禁止值错误（红标 🔴）

**触发条件**：任何通道出现 `000h` 或 `3FFh`

```
错误类型: forbidden
显示颜色: #ff6b6b (红色)
严重程度: 最高
```

这是 BT.656 标准明确禁止的数据值，通常表示：
- 数据传输错误
- 同步丢失
- 硬件故障

### 2. 越界错误（橙标 🟠）

**触发条件**：
- Y 通道：`< 040h` 或 `> 3BFh`（且 ≠ 000h/3FFh）
- Cb/Cr 通道：`< 040h` 或 `> 3C0h`（且 ≠ 000h/3FFh）

```
错误类型: out_of_range
显示颜色: #ffa500 (橙色)
严重程度: 中等
```

越界值可能导致：
- 画面过亮/过暗
- 色彩失真
- 显示设备处理异常

### 3. 阶梯错误（黄标 🟡）

**触发条件**：值在合法范围内，但不符合 SMPTE 彩条标准阶梯

```
错误类型: staircase
显示颜色: #ffd700 (黄色)
严重程度: 较低（仅当启用 SMPTE 阶梯检测时触发）
```

典型场景：
- Y 值为非标准黑电平（如 `010h`, `020h`）
- Cb/Cr 为异常低值（如 `008h`, `010h`, `012h`）
- 彩条电平与 SMPTE 75% 标准偏差超容限

### 合法范围速查表

| 通道 | 最小值 | 最大值 | 禁止值 |
|------|--------|--------|--------|
| Y | 040h (64) | 3BFh (959) | 000h, 3FFh |
| Cb | 040h (64) | 3C0h (960) | 000h, 3FFh |
| Cr | 040h (64) | 3C0h (960) | 000h, 3FFh |

## 模块详解

### 1. `server/bt656-engine.js` — 核心解析引擎

**职责**：纯算法模块，无外部依赖，可独立测试或集成到其他项目。

#### 常量定义

```javascript
// SAV/EAV 控制字集合
const SAV_EAV_WORDS = new Set([0x200, 0x20D, 0x27C, 0x279, 0x1B4, 0x1B1, 0x168, 0x165])

// SMPTE 75% 彩条参考电平 (10-bit Y值)
const SMPTE_REF_Y = [0x3AC, 0x340, 0x2DC, 0x270, 0x21C, 0x1B0, 0x154, 0x040]
// 对应名称: ['白', '黄', '青', '绿', '品红', '红', '蓝', '黑']

// 合法范围边界
const Y_MIN = 0x040, Y_MAX = 0x3BF      // Y: 64 ~ 960
const C_MIN = 0x040, C_MAX = 0x3C0      // Cb/Cr: 64 ~ 960
const FORBIDDEN_LO = 0x000, FORBIDDEN_HI = 0x3FF  // 禁止值

// 行时序参数 (BT.656 标准)
const LINE_TIMING = {
  720:  { totalPerLine: 858,   eavPosition: 720,  savPosition: 800 },
  1280: { totalPerLine: 1650,  eavPosition: 1280, savPosition: 1360 },
  1440: { totalPerLine: 1980,  eavPosition: 1440, savPosition: 1520 },
  1920: { totalPerLine: 2200,  eavPosition: 1920, savPosition: 2000 },
}
```

#### 核心函数

| 函数 | 输入 | 输出 | 功能 |
|------|------|------|------|
| `unpack10bitPacked(rawBytes)` | Uint8Array | number[] | 将 5 字节解包为 4 个 10-bit 样本 |
| `unpack10bitRaw(rawBytes)` | Uint8Array | number[] | Raw 10-bit 模式解包 |
| `detectSAV_EAV(samples)` | number[] | Marker[] | 检测 `3FF 000 000 XYZ` 同步头 |
| `extractActiveVideo(samples, markers, lineWidth)` | samples, markers, width | Region[] | 提取有效视频区域 |
| `checkRange(val, channel)` | value, 'Y'/'Cb'/'Cr' | {outOfRange, reason} | 范围越界判断 |
| `analyzeStaircase(yValues, tolerance)` | Y值数组, 容限 | StairResult | SMPTE 8级阶梯分析 |
| `analyze(rawData, options)` | 原始字节, 配置对象 | AnalysisResult | **主分析入口** |

#### `analyze()` 主流程

```
输入: rawData (Uint8Array), options: { width, tolerance, packMode, initialOffset }
输出: { stats, errors, stairResults, markers, regions, rawData, allSamples }

┌──────────────────────────────────────────────────────────────┐
│ 1. 解包 10-bit 样本                                          │
│    unpack10bitPacked() 或 unpack10bitRaw()                   │
└───────────────────────────┬──────────────────────────────────┘
                            ▼
┌──────────────────────────────────────────────────────────────┐
│ 2. 检测 SAV/EAV 同步头                                       │
│    detectSAV_EAV() → markers[]                               │
│    解析 F/V/H 位，判断场/消隐状态                             │
└───────────────────────────┬──────────────────────────────────┘
                            ▼
┌──────────────────────────────────────────────────────────────┐
│ 3. 提取有效视频区域                                          │
│    extractActiveVideo() → regions[]                          │
│    每个 region = { startSample, endSample, lineNumber, ... } │
└───────────────────────────┬──────────────────────────────────┘
                            ▼
┌──────────────────────────────────────────────────────────────┐
│ 4. 遍历每个区域，按 Cb→Y→Cr→Y 循环处理                       │
│    for (i = 0; i < len; i += 4) {                            │
│      cb = samples[i], y0 = samples[i+1],                     │
│      cr = samples[i+2], y1 = samples[i+3]                    │
│                                                              │
│      ├─ checkRange(cb, 'Cb')  → 越界则记录错误               │
│      ├─ checkRange(y0, 'Y')   → 越界则记录错误               │
│      ├─ checkRange(cr, 'Cr')  → 越界则记录错误               │
│      └─ checkRange(y1, 'Y')   → 越界则记录错误               │
│    }                                                         │
└───────────────────────────┬──────────────────────────────────┘
                            ▼
┌──────────────────────────────────────────────────────────────┐
│ 5. SMPTE 阶梯分析 (每行)                                     │
│    analyzeStaircase(lineY[], tolerance)                      │
│    ├─ 将 Y 数组等分为 8 区间                                  │
│    ├─ 计算每区间均值、标准差                                  │
│    ├─ 与 SMPTE_REF_Y 参考值比对                              │
│    └─ 偏差超容限 → 标记为阶梯错误                             │
└───────────────────────────┬──────────────────────────────────┘
                            ▼
┌──────────────────────────────────────────────────────────────┐
│ 6. 计算样点序号                                              │
│    if (initialOffset !== 0) {                                │
│      // 手动偏移模式                                          │
│      baseIndex = initialOffset + (lineNum-1) * totalPerLine  │
│    } else {                                                  │
│      // 自动模式：基于 BT.656 标准时序                        │
│      baseIndex = (lineNum-1) * totalPerLine + savPosition+4  │
│    }                                                         │
└──────────────────────────────────────────────────────────────┘
```

## 核心算法

### 10-bit Packed 解包算法

BT.656 标准的 4:2:2 打包格式：**每 5 字节存储 4 个 10-bit 样本**

```
字节布局:
  Byte[0]: Sample0[9:2]   (高8位)
  Byte[1]: Sample1[9:2]
  Byte[2]: Sample2[9:2]
  Byte[3]: Sample3[9:2]
  Byte[4]: Sample0[1:0] | Sample1[1:0] | Sample2[1:0] | Sample3[1:0]
          (低2位拼接: 8 bits = 2+2+2+2)

解包公式:
  Sample0 = (Byte[0] << 2) | (Byte[4] >> 6)
  Sample1 = (Byte[1] << 2) | ((Byte[4] >> 4) & 0x03)
  Sample2 = (Byte[2] << 2) | ((Byte[4] >> 2) & 0x03)
  Sample3 = (Byte[3] << 2) | (Byte[4] & 0x03)
```

### SAV/EAV 同步头检测

BT.656 定时基准信号格式：`3FF 000 000 XYZ` (4 个 10-bit 字)

```
XYZ 控制字结构 (10-bit):
  ┌─────────────────────────────────────────────┐
  │ 1  F  V  H  P3 P2 P1 P0 0 0 │
  │ ↑  ↑  ↑  ↑  └──────────────┘ │
  │ │  │  │  │    校验位 (Hamming)│
  │ │  │  │  └─ H: 水平消隐 (0=EAV, 1=SAV)│
  │ │  │  └─ V: 垂直消隐│
  │ │  └─ F: 场标识 (0=Field1, 1=Field2)│
  │ └─ 固定为 1│
  └─────────────────────────────────────────────┘

常见 XYZ 值:
  SAV (有效视频开始): 0x200, 0x20D, 0x27C, 0x279
  EAV (有效视频结束): 0x1B4, 0x1B1, 0x168, 0x165
```

### 样点序号计算

**自动模式** (initialOffset = 0):
```
第 N 行起始序号 = (N-1) × totalPerLine + savPosition + 4

例: 1440×1080i, 第1行 → 0 × 1980 + 1520 + 4 = 1524
```

**手动偏移模式** (initialOffset ≠ 0):
```
第 N 行起始序号 = initialOffset + (N-1) × totalPerLine

例: initialOffset=1437, 第1行 → 1437
initialOffset=1437, 第2行 → 1437 + 1980 = 3417
```

### 三、功能模块详解

#### 模块 1：数据输入

| 功能 | 说明 |
|------|------|
| **输入模式切换** | 选择"流数据"或"CSV/Excel"两种分析模式 |
| **文件上传** | 支持拖拽或点击上传 .csv / .xlsx 文件 |
| **WFM8300 格式** | 自动识别泰克 WFM8300 导出的数据格式 |

**操作步骤**：

2. 拖拽文件到上传区域，或点击"选择文件"

#### 模块 2：解析配置

| 参数 | 说明 | 可选值 |
|------|------|--------|
| **分辨率** | 视频每行 Y 样点数 | 自动检测 / 720 / 1280 / 1440 / 1920 |
| **起始序号偏移** | 样点序号起始值 | 0（自动）或手动输入（如 1437） |
| **SMPTE 阶梯检测** | 是否启用彩条阶梯校验 | 开启 / 关闭 |
| **阶梯容限** | 允许的电平偏差值 | 1~20 LSB（默认 5） |

**使用场景**：
- **自动模式**：适用于完整视频帧数据
- **手动偏移**：适用于视频流片段（如行尾消隐区），输入起始序号可校准显示

#### 模块 3：WFM8300 文件信息

| 总Y样点/行 / 有效Y样点/行 | 每行的样点结构 |
| **Events at Trigger** | 触发时的事件计数（EAV Error、SAV Error 等） |

> 💡 点击"WFM8300 文件信息"标题栏可折叠/展开详细信息

### 四、错误判定规则

#### 合法范围

| 通道 | 最小值 | 最大值 | 说明 |
|------|--------|--------|------|
| **Y (亮度)** | 040h (64) | 3BFh (959) | 黑电平 ~ 白电平 |
| **Cb / Cr (色度)** | 040h (64) | 3C0h (960) | 色度范围 |

#### 特殊值

| 值 | 含义 | 判定 |
|----|------|------|
| **000h** | SAV/EAV 定时基准 | ✅ 合法（不报错） |
| **3FFh** | SAV/EAV 定时基准 | ✅ 合法（不报错） |

#### 错误类型

| 类型 | 触发条件 | 颜色标识 |
|------|----------|----------|
| **越界错误** | Y < 040h 或 > 3BFh<br>Cb/Cr < 040h 或 > 3C0h | 🟠 橙色 |
| **阶梯错误** | 值在合法范围内，但不符合 SMPTE 彩条标准阶梯 | 🟡 黄色 |

### 五、常见使用场景

#### 场景 1：分析 WFM8300 导出的 CSV 文件

1. 选择"CSV/Excel"输入模式
2. 上传从 WFM8300 导出的 .csv 文件
3. 展开"WFM8300 文件信息"查看视频格式和事件
4. 在"数据浏览"中逐行检查数据
5. 如有错误，切换到"异常详情"查看具体问题

#### 场景 2：分析视频流片段（非完整帧）

1. 选择"流数据"输入模式
2. 上传 .bin / .raw 文件或粘贴 Hex 数据
3. 在"解析配置"中设置：
   - 分辨率：选择对应的视频格式
   - 起始序号偏移：输入片段的起始样点号
4. 点击"开始分析"

#### 场景 3：定位特定行的错误

1. 上传数据文件
2. 切换到"行统计"标签页
3. 找到错误率较高的行
4. 点击"查看"按钮展开该行详情
5. 在样点详情中定位具体错误样点
